## 1. 总体结论

**这个项目能不能进一步优化？能，而且有两类性质完全不同的问题，必须分开看：**

### 1.1 关键背景：edge 研究目前的结论是「无 edge」
测绘显示：`docs/FINDINGS_v2.md` 与前向协议记录，活跃的日内/多周期 edge 研究**已得出 NO TRADE / NO EDGE 判决**。系统当前的真实价值是「一套可复现的负结果方法学 + PIT 数据层」，**不是一个可盈利的实盘策略**。这一点决定了风险的优先级排序。

### 1.2 风险是**不对称**的 —— 这是本次审核最重要的洞察
- **研究有效性的几乎所有缺陷都偏向"制造假阳性 PASS"**：非粘性 KILL、多重检验按 3 个而非 7 个校正、ST720 事后挑板块、活跃判决门不调 DSR/PBO、卖方策略零价差成本……这些**不会**让一个真负结果变正，但**会**把一个本该死掉的候选"复活"成 PASS。
  → **当前结论是"无 edge"，所以这些缺陷此刻没有造成损失；但它们正是"未来某天你看到一个 PASS 并据此投入真金"时，会凭空制造那个 PASS 的元凶。** 在相信任何 PASS 之前必须修。
- **资金安全/执行的缺陷则是"此刻就会亏钱"**：只要打开 `dist\OKXB.exe` 并勾选实际下单，下面的 7 个 critical 与多个 high **与 edge 无关、立即生效**。其中最危险的是**打包版 .exe 跑的是和你审计的源码完全不同的、激进约 2 倍的配置**，以及**崩溃后持仓在交易所端没有任何止损**。

**一句话**：研究方法学需要"防自己骗自己"的加固（防假 PASS）；实盘执行层需要"防真金失控"的加固（防裸仓 / 防配置漂移 / 防硬停失效）。前者关乎可信度，后者关乎本金。

### 3.2 剔除 / 降级的发现（透明记录 —— 验证的价值就在这里）

| 原发现 | 处置 | 读代码后的理由 |
|---|---|---|
| stat[8]「横截面多空净edge只计 1 腿成本」 | **剔除（误报）** | `evaluate_scores`（pmw:366）是**逐标的方向性**仓位（sign(score)×fwd 等权），单标的=1 个往返=1 腿，计 1 腿正确；代码在**真正多空**处已用 `n_legs=2`（pmw:437）。审核员把"逐标的方向"误读成"两腿多空书"。 |
| stat[12]「purged WF embargo 只等于 H」 | **降级 → 可忽略** | 前向链式划分（train≤cut-H，test>cut），`embargo_ms=H` 已正确清除标签重叠；特征是 trailing，不会把 test 信息漏进 train。 |
| stat[1]「横截面把相关币当独立样本」 | **降级 → low** | 主判决指标 `primary_ic` 用的是**逐时戳横截面 IC + NW-t**（正确）；只有辅助的 pooled AUC/winrate 高估了有效 N，而它们明确标注为"仅参考"。 |
| subproj[6]「冻结缺 model.pkl」 | **剔除（误报）** | `frozen/A/model.pkl` 等**实际存在于磁盘**；"tracked 的只有 meta/feature/median"也不对（**没有任何文件被 git 跟踪**）。该问题已被 RV-1 完整覆盖。 |
| leak[1]「DVOL `publish_lag=0` 跨日前视」 | **降级并入 RV-8** | 关键发现：`inst_bank` 把行锚在 bar-**open** ts，但特征用 bar-**close**、`fwd` 也从 close 起算 → 价格特征与日频外部其实**都对齐到 bar-close**、互相一致，因此**未证明存在经济意义上的跨日前视**；真正问题是 PIT 纪律被绕过（RV-6/RV-8）而非确凿泄漏。 |

> **结论修正**：审核员把 leak/cost 几条标成了 HIGH 跨日泄漏，但读码后多为"纪律/可审计性"问题而非确凿经济泄漏。这降低了"既有研究结论被泄漏污染"的概率——**对一个"无 edge"的负结论而言，这进一步说明该负结论是稳健的**（泄漏只会夸大正，不会制造负）。
